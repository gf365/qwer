Profile

QWER TOKYO は、AutoHanko のような実用アプリを一人で形にする個人プロジェクトです。

企画、情報設計、UI、実装、公開時の見せ方までを分けずに扱い、動くアプリとして外に出せるところまでまとめて作ります。現在の公開例が、PDF作業を短くする電子捺印アプリ AutoHanko です。

Strength

得意なのは、面倒な作業を短い体験へ整理することです。

作業フローの整理

複数ステップにまたがる操作でも、入口と終点を明確にして、途中で止まりにくい画面へまとめます。

初回利用の分かりやすさ

ガイド、モード切替、段階的なUIを使い、いきなり複雑に見えない状態から始められるようにします。

公開導線まで含めた設計

アプリ本体だけでなく、サイト上の紹介、詳細ページ、外部リンクまで含めて実物へつなげます。

Working Style

まず動くものを公開し、そこから説明と導線を整えます。

今回のサイト更新も同じ考え方です。抽象的なテンプレート紹介を残すのではなく、AutoHanko の実画面と公開リンクを前に出し、何が使えるのかを先に伝える構成へ寄せました。

Live App First Workflow Oriented Clear UI Public Demo Ready

Values

制作で重視していること

  • 何の作業を短くするアプリかを最初に伝えること。
  • 初回利用者が次の操作を迷わないこと。
  • 実装物と紹介サイトの内容がずれないこと。
  • 公開後に改善しやすい構造で出すこと。

Open / Contact
